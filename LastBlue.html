The Last Blue

The Last Blue was our third Game project in Futuregames and it was made by a team of 16 people.
In the game, you play as a young girl who finds herself lost within an ancient shrine. She realizes that to escape the shrine, she will have to learn the language that seems to be scribbled on the walls and start understanding what it means.

She encounters a Shrine Spirit who guides her with learning the language. The spirit also provides her with a magic tool with which she can modify the languages written on the walls.

You traverse the three floors of the shrine, and as you explore, you will have to solve many puzzles that will make use of your knowledge of this language. Subsequently, you will learn the story of the people and the civilization that existed here, as well as the mystery behind this Shrine Spirit.

AI | UI

My main resposibility for The Last Blue was implementing the AI. It uses Unreal Engine's Behavior Tree.
The AI is friendly and is designed to guide the player throughout the game. It performs a variety of actions, based on:

- The player's position
- The AI's distance from the player
- Game progression


I also did the menu UI and game settings.
The menu UI includes, an options menu to change game settings, as well as key-bindings.

EXPLORE TASK

The Explore Behavior is similar to an idle state. The AI will randomly move around in the proximity of the player
The Task sequence is:

Clear Focus - Removes the current target.
Find Location - Get a random position around the player.
Move - Takes the shortest path to the target.
Wait - Stands still for a random range of time before starting a new sequence.

FOLLOW TASK

The Follow Behavior makes the AI follow the player.
The sequence of the task is:

Clear Focus - Removes the current target.
Move - Takes the shortest path to the player.
Focus - Looks at the player.
Wait - Pause for a second to not instantly resume following.

WAYPOINT TASK

The Waypoint Behavior gets a set position for the AI to move to.
The sequence of the Task is:

Clear Focus - Removes the current target.
Find Waypoint - Gets a specific postion from a predefined list of waypoints.
Move - Takes the shortest path to the waypoint.
Set State - Updates to a new AI state based on the next intended action.

WAIT TASK

The Wait Behavior is an idle state. I makes the AI stand still.
There are two actions in the sequence:

Clear Focus - Removes the current target.
Focus - Sets player as the target.

OUT OF RANGE TASK

The Out of Range Behavior makes the AI stay within a certain distance to the player, and not run to far ahead.
The Task sequence is:

Clear Focus - Removes the current target.
Wait - Pause for a short moment.
Move - Takes the shortest path to the player until a set threshold is reached.
Wait - Pause for a short moment.
Focus - Looks at the player until they reach a set threshold and catch up.

TALK TASK

The Talk Behavior is used to show the player glyphs to learn the language.
The talk sequence goes:

Clear Focus - Removes the current target.
Face Player - Moves in front of player, keeping a set distance between them.
Focus - Sets player as the target.
Talk - Displays a glyph.
Wait - Waits for a short moment.
Stop Stops displaying glyph.
Set State Updates to a new AI state based on the next intended action.

SHOW GLYPH TASK

The Show Glyph Behavior is also used to show the player glyphs. The difference is, the AI displays the glyph from its current position, instead of in front of the player.
The sequence of the task is:

Clear Focus - Removes the current target.
Focus - Sets player as the target.
Talk - Displays a glyph.
Wait - Waits for a short moment.
Stop Stops displaying glyph.
Set State Updates to a new AI state based on the next intended action.
